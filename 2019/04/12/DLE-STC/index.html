DLE

论文：A double layered ‘plus-minus one’ data embedding scheme(2007,SPL,CR86)

  对于一般的二元编码，一般以灰度图像的LSB位为二元载体序列进行嵌入。事实上，在LSB位进行嵌入时，有两种修改方式，一种是直接LSB替换，另外一种是根据LSB位对像素灰度值随机±1。因为加1或减1操作都可以修改LSB位，但通过选择加1或减1可以控制次LSB位的取值，所以能够同时在次LSB位嵌入消息，而不需引入额外修改。但是由于接收者无法知道哪些像素需要±1修改，所以第二层嵌入需要使用湿纸编码，这一方法称为双层嵌入。

  双层隐写编码的具体实现过程如下:用L(g)表示像素g=（g1,g2,g3,…,gn）的LSB层，用S(g)表示像素的次LSB层。首先，在LSB层L(g)使用二元编码(如矩阵编码)G嵌入消息，设编码G的嵌入率为a，修改率为c。则第一层可嵌入na，平均有nc个像素的LSB位需要修改，但在这一步并不作修改，只是标记需要修改的像素;然后，在次LSB层S(g)嵌入消息，将第一步中被标记像素的次LSB位定义位“干点”，其他像素的次LSB位定义为“湿点”。这里设湿纸码的嵌入率为b.，则使用湿纸码平均可以嵌入ncb比特消息，这一步可以确定次LSB位的取值。根据被标记像素的次LSB的取值，可以确定对这些像素应该采用加1还是减1的方式进行修改。

  上述方法，在n长载体中共嵌入na+nc比特消息，平均需要修改nc个像素，所以其嵌入率为a+c，而修改率仍为c。若在LSB层以最大嵌入率1进行嵌入，则平均有1/2的像素需要修改，此时，双层嵌入的最大嵌入率为1+0.5b。

STC

论文：Minimizing additive distortion functions with non-binary embedding operation in steganography(2010,WIFS,CR14)

Minimizing additive distortion in steganography using syndrome-trellis codes(2011,TIFS,CR180)

Minimizing embedding impact in steganography using trellis-coded quantization(2010,SPIE,CR108)

  STC编码方法可以在湿度级模型下逼近嵌入效率上界。STC码也是一种特殊的矩阵编码，所用的矩阵H由一个hxw子矩阵以一种级联的方式拼接而成。这种构造使得Hy =m的每个解都可以表示成通过栅格的一条路径，路径每一步的长度由湿度级定义，从而最小化嵌入所引起的失真就变成了寻找最短路径问题，而最短路径可以由维特比译码的方式快速得到。编码的嵌入率为1/w，w的值取决与期望负载，若负载α=1/k（k∈N）则w=k，若1/（k+1）≤α≤1/k，则w=k+1。由子矩阵的宽决定。而子矩阵的高h决定了每个载体比特对应的路径条数为2h，因而h越大最小化失真的能力越强，但编码的计算复杂度越高，一般来说h∈[6,15]。

维特比算法

目前已经写了两篇文章，十分形象讲解了维特比算法的过程。

Gibbs分布

  满足给定约束条件且熵最大的分布。

  介绍：https://blog.csdn.net/hdt1720/article/details/21240415

  Jessica Fridrich关于Gibbs分布的论文：http://dde.binghamton.edu/filler/pdf/Fill10tifs-gibs-journal.pdf
对于一个给定的嵌入失真函数，当我们嵌入最大负载时，嵌入引起的变化遵循Gibbs分布。这篇论文的独特之处是它允许失真函数是任意的， 这使得我们可以考虑空域独立嵌入引起的变化。它提供了一个完整的理论框架来描述实用工具，其中就有说到使用Gibbs Sampler来模拟最优嵌入方案的影响，从而构造实用的算法。该框架将在有经验的cover下的安全隐写术的设计简化成为了失真函数求解问题，十分有用。

STC编码

  STC 是基于伴随式编码( syndrome coding) 的隐写编码方法。设秘密消息 m 长为Lm，载体 c 长为 Lc，STC 将秘密信息看作一个长度为 Lc 的线性码 ζ 的伴随式，ζ由校验矩阵 D1 唯一确定。m 的陪集 C( m) 定义如下:

C(m)={z∈{0，1}Lc| D1z=m}

  STC就是在C(m)中寻找相对c累积失真最小的 Lc长序列，将其作为隐秘载体输出。它的编码是在网格上实现的，网格内顶点表示可能出现的伴随式样式;边对应 Lc 长序列中的元素(在二元域内，元素的值非 0 即 1) 。网格内的一条路径表示C(m)中的一个序列。当根据失真度量方式为边设置权重后，编码过程就可以找到累积权重最小的路径，即失真最小的隐秘载体。其中失真度量方式由研究者根据载体特性确定。

湿纸码提高STC嵌入效率

  如果把 STC 方法应用到湿纸编码信道上，即某些点为 wet点，其代价应该为无穷大，是不能被修改的。如果嵌入率为h，湿纸率为w，则隐秘信息长度m由之前的αn变成了(1-w)αn，H大小变成了(1-w)αn*n。但是在实际应用STC 方法的时候，wet 点的修改还是很难避免的，总会有几个 wet点不得不被修改，从而导致嵌入失败。

  针对这种问题，这有两种处理思路。

  1、不修改任意一个 wet点。由于嵌入失败的概率P与α，w，h有关，但这只是个概率而已，并不一定就会修改wet点，而具体的 wet 点被不被修改，与 cover 的比特顺序有关，即在某一种 cover比特顺序情况下，wet 点可能会被修改，而在另一种 cover 比特顺序情况下，wet点有可能就不会被修改了。所以在P一定的前提下，用密钥随机置乱 cover比特，多试几次，使 wet 点碰巧不被修改。这个密钥有 k位，在图像中专门找一个区域用于存放。那么最后 wet 点仍面临修改的概率为P2k，几乎为0。在 TIFS 版本中，这 k位密钥用隐秘信息长度m来表示，不断减小m进行试验。在实际的实验中，cover图像有10的6次方个像素，湿纸率w=0.8，h=10，1000次独立运行后，若α取1/10，P即可等于0。

  2、假设修改几个wet 点对 stego 的安全性影响不大，给 wet点分配一个足够大但非无穷大的代价（该代价大于所有 dry点的代价之和），使得最后只修改少数几个 wet点而不影响整体的安全性。下图展示的就是在10的6次方像素图像中对于相关给定的负载我们平均需要修改的wet元素个数。

多进制应用

  尽管将STC扩展到非二进制并将其应用于多进制嵌入操作非常简单，但是它们的复杂性会迅速增加(对于约束高度，格子中的状态数从到增加)，从而在实践中限制了它们的性能。，论文提出了一个简单的分层结构，它主要是由根据我们前面学到的双层嵌入（DLE）思想提出的，可以看作是这项工作的概括。其主要思想是将使用非二进制嵌入操作的问题PLS和DLS分解为使用二进制嵌入操作的类似问题序列。任何解决二进制PLS嵌入问题的方案，如STC，都可以使用。如果每个二进制嵌入问题都得到最优解，那么这种分解就是最优的。
