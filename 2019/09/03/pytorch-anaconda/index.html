pytorch+anaconda

发表于 2019-09-03

记录一下大致碰到的问题。

  1. 因为当初用anaconda安装tensorflow，它不支持python3.7版本。所以anaconda选择比较老一点的版本Anaconda3-5.2.0-Windows-x86_64。

  2. 按照网上的流程安装好Anaconda。

  3. 安装CUDA，cudnn。并配置好相关的环境变量。

  4. Anaconda安装Pytorch。

在官网（https://pytorch.org/get-started/locally/）下载系统相应版本安装。

为了防止其他的环境之间影响，先创建一个环境，命名为pytorch：

conda create -n pytorch python=3.6

然后切换到该环境：

activate pytorch

然后根据官网的内容：

如上图，根据相应的环境执行对应的语句，例如我的是：

conda install pytorch torchvision cudatoolkit=10.0 -c pytorch

安装Pytorch过程中会连接失败，这是因为Anaconda.org的服务器在国外。网上说的使用清华，中科大的镜像并没有用，已经都失效了，更改了镜像库也没有用。所以直接不要改。

直接看报错信息，可以看到我们是下载pytorch和torchvision失败了。

报错信息中给出了对应的连接下载超时的地址，这时候只需要我们翻墙然后输入对应的网址

(torchvision)https://conda.anaconda.org/pytorch/win-64/torchvision-0.4.0-py36_cu100.tar.bz2

(pytorch)https://conda.anaconda.org/pytorch/win-64/pytorch-1.2.0-py3.6_cuda100_cudnn7_1.tar.bz2

即可下载对应的两个源文件。

然后在对应的文件夹中对两个文件进行离线安装，即可完成对应的pytorch安装。

检验的时候发现缺少了部分依赖（numpy，pillow，six），报错缺少什么直接pip安装什么即可（或者直接执行pip
install torchvision即可自动安装所有的必备的依赖）：
